# JTC Excel設計書 AIナレッジ化 PoC

## 顧客課題

大手企業の設計書は Word/Excel、とくに Excel方眼紙形式で蓄積されている。セルの値だけでなく、罫線、結合セル、色、入力規則、コメント、図形に仕様情報が埋め込まれているため、一般的な Markdown 変換では意味構造が失われる。

## PoCで検証すること

- Excel方眼紙の罫線領域を表・セクションとして抽出できるか
- 結合セルのタイトル/セクション見出しを保持できるか
- 入力規則やコメントなど、レビューが必要な仕様情報を warnings に出せるか
- MarkdownとJSONを併出し、人間レビューとRAG投入の両方に使えるか
- 人手でのMarkdown化作業時間を50%以上削減できるか

## MVP成果物

```text
outputs/<document-name>/
  extracted.json      # 構造化抽出データ
  specification.md    # 人間/AI向けMarkdown
  warnings.md         # 要確認箇所
```

## 想定スコープ

- 対象: 1部署、Excel設計書 10〜30ファイル
- 期間: 2〜4週間
- 価格目安: 50万〜150万円
- 次フェーズ: SharePoint/Box/Git連携、社内RAG、要件定義・テストケース生成への展開

## 技術方針

1. `openpyxl` でセル値・罫線・結合セル・色・入力規則・コメントを抽出する。
2. 罫線付きセルの連結成分から、表/ブロック候補を検出する。
3. MarkdownとJSONを出力する。
4. 変換不能または人手確認が必要な情報は warnings に明示する。
5. LLM整形は次段階。まずは構造抽出を決定論的に行う。
